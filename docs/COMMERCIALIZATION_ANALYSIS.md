❌ 결제 시스템 - Stripe, Toss 등 미연동

❌ 멀티 디바이스 동기화 - 단일 API 서버

## 2. 시장 분석

### 2.1 잠재 경쟁 제품 비교

| 제품 | 가격 | 핵심 가치 | Aletheia 대비 |
|------|------|-----------|---------------|
| **Notion AI** | $10/월 | 범용 AI 어시스턴트 | 기능 풍부, 생태계 완성 |
| **Reflect** | $15/월 | AI 기반 저널링 | 유사 컨셉, 선점 효과 |
| **Day One** | $3-5/월 | 일기 앱 | 단순하지만 완성도 높음 |
| **Roam Research** | $15/월 | 그래프 기반 노트 | 가치 그래프와 유사 |
| **ChatGPT Plus** | $20/월 | 범용 AI 상담 | "그냥 물어보면 됨" |

### 2.2 핵심 문제: ChatGPT와의 경쟁

```
사용자 질문: "이직할까 말까?"

ChatGPT 응답 (즉시, 무료~$20):
  → 장단점 분석
  → 고려사항 제시
  → 질문을 통한 탐색

Aletheia 응답:
  1. 먼저 수십 개의 생각 파편을 입력해야 함 (수주~수개월)
  2. 그 후에야 "62% 확률로 A에 적합" 제시
  3. 추천은 안 함 (철학적 이유로)
```

**치명적 약점**: 가치를 느끼려면 오랜 기간의 투자가 필요하나, 그 전에 이탈할 가능성이 높음

### 2.3 타겟 시장 문제

| 잠재 타겟 | 규모 | 문제점 |
|-----------|------|--------|
| 결정 장애 있는 사람 | 소규모 | 심리 상담 서비스로 해결 |
| 자기 성찰 매니아 | 극소규모 | 이미 자신만의 방법 보유 |
| 일반 직장인 | 대규모 | Notion, 캘린더로 충분 |
| 코칭 받는 사람 | 중간 | 코치가 직접 가이드 |

## 3. 가치 제안 분석

### 3.1 현재 가치 제안

```
"과거의 나를 기반으로 현재 결정의 적합도를 확률로 제시"
```

#### 문제점

1. **추상적**: "나다운 결정"이 무엇인지 체감하기 어려움
2. **수동적**: LLM이 추천을 안 하면 "그래서 뭘 하라는 거야?"
3. **장기 투자 필요**: 데이터 축적에 수개월 필요
4. **확률의 함정**: 62% vs 38%가 실질적 결정에 도움이 되는가?

### 3.2 경쟁사 대비 차별점

| 차별점 | 장점 | 단점 |
|--------|------|------|
| 추천 안 함 | 철학적 순수성 | 사용자 기대와 불일치 |
| 수학적 계산 | 재현 가능성 | LLM 대비 유연성 부족 |
| 가치 그래프 | 자기 이해 도구 | 시각화 없이는 무의미 |
| 모순 허용 | 현실 반영 | 혼란을 줄 수 있음 |

### 3.3 "왜 돈을 내야 하나?"에 대한 답변 부재

```
❌ "메모 저장" → 무료 앱 많음
❌ "AI 분석" → ChatGPT로 가능
❌ "결정 도움" → 추천을 안 해줌
❌ "자기 이해" → 가치 그래프 시각화 없음
❌ "감정 추적" → 전문 앱 존재 (Daylio, Moodpath)

✅ 유일한 차별점: "과거의 나 기반 확률 계산"
   → 이것만으로 월 $10+ 구독을 유도할 수 있는가? 의문
```

## 4. 기술적 리스크

### 4.1 운영 비용 분석

```kotlin
// 1회 Fragment 생성 비용
embedding: ~$0.0001 (text-embedding-3-small, 1K tokens)
emotion_analysis: ~$0.001 (gpt-4o-mini)
value_extraction: ~$0.001 (optional)
// 소계: ~$0.002/fragment

// 1회 Decision 생성 비용
context_embedding: ~$0.0001
explanation: ~$0.01 (gpt-4o-mini, 1K output)
// 소계: ~$0.01/decision

// 활성 사용자당 월간 비용 추정
30 fragments × $0.002 = $0.06
5 decisions × $0.01 = $0.05
// 합계: ~$0.11/user/month (API 비용만)
```

| 비용 항목 | 월간 비용 (1,000 사용자 기준) |
|-----------|-------------------------------|
| OpenAI API | ~$110 |
| PostgreSQL (managed) | $50-200 |
| 서버 (Kubernetes/EC2) | $100-300 |
| 총 운영비 | **$260-610** |

**최소 구독료**: $3-5/월 이상 필요 (마진 포함 시)

### 4.2 확장성 우려

```
[테스트 부재]
- 91개 소스 파일 중 3개만 테스트
- 벡터 검색 성능 미검증
- 동시 접속 부하 테스트 없음

[잠재적 병목]
- HNSW 인덱스: 100만 레코드 이상 시 성능 저하 가능
- LLM 호출: Rate limit, 응답 지연
- 단일 DB: 수평 확장 어려움
```

## 5. 상용화를 위한 개선 방안

### 5.1 즉시 필요 (Phase 1: 0-3개월)

| 우선순위 | 항목 | 예상 공수 | 이유 |
|----------|------|-----------|------|
| 🔴 Critical | 프론트엔드 개발 | 2-3개월 | API만으로는 사용자 없음 |
| 🔴 Critical | 가치 그래프 시각화 | 2-4주 | 핵심 차별점 체감 필요 |
| 🔴 Critical | 온보딩 플로우 | 1-2주 | 첫 경험이 구독 전환 결정 |
| 🟡 High | 테스트 코드 확충 | 3-4주 | 안정성 확보 |
| 🟡 High | 모바일 최적화/앱 | 2-3개월 | 일상 기록은 모바일 필수 |

### 5.2 가치 제안 재정의 (Phase 2: 3-6개월)

#### Option A: "AI 코치" 방향으로 피벗

```diff
- "추천 안 함" 원칙 완화
+ "참고 의견" 형태로 제안 추가
+ 사용자가 원할 때만 조언 활성화

장점: 시장 기대에 부합
단점: 철학적 정체성 상실
```

#### Option B: "정량적 자기 이해" 강화

```diff
+ 가치 그래프 트렌드 리포트 (주간/월간)
+ "나의 가치관 변화" 인사이트
+ 결정 이력 대시보드
+ PDF 리포트 생성

장점: 차별화 유지
단점: 여전히 틈새 시장
```

#### Option C: B2B 피벗 (코칭/상담 도구)

```diff
+ 코치/상담사 대시보드
+ 클라이언트 가치 프로파일 공유
+ 세션 전 사전 분석 리포트

장점: 높은 단가 ($50-200/seat)
단점: 영업 필요, 긴 판매 주기
```

### 5.3 수익 모델 제안

#### Freemium 모델

| 플랜 | 가격 | 제한 |
|------|------|------|
| Free | $0 | 월 30 fragments, 3 decisions |
| Pro | $9.99/월 | 무제한, 가치 리포트 |
| Coach | $29.99/월 | Pro + 클라이언트 관리 (B2B) |

#### 주의사항

```
❌ 피해야 할 것:
- Fragment 수 기반 과금 → 사용 억제
- Decision 수 기반 과금 → 핵심 기능 제한

✅ 권장:
- 분석 깊이/리포트 기반 차등
- 데이터 보관 기간 차등
- API 접근 (개발자용)
```

## 7. 결론 및 권고

### 7.1 냉정한 평가

```
현재 상태의 aletheia-core는:

1. 기술적으로는 MVP 수준 완성 (7/10)
2. 상용화 준비도는 매우 낮음 (2/10)
3. 시장 검증이 전혀 되지 않음 (0/10)

핵심 문제:
"왜 사용자가 이 서비스에 돈을 내야 하는가?"에 대한
명확한 답변이 없음
```

### 7.2 권고 사항

#### 즉시 실행

1. **프론트엔드 개발 착수** - API만으로는 사용자 확보 불가
2. **10명의 베타 테스터 확보** - 실제 사용 패턴 파악
3. **가치 그래프 시각화** - 핵심 차별점 체감 필요

#### 3개월 내

4. **온보딩 최적화** - 첫 5분 내 가치 체감 설계
5. **빠른 가치 제공** - 3개 fragment만으로도 인사이트 제공
6. **결제 시스템 연동** - Stripe/Toss 연동

#### 결정 필요

7. **철학적 방향성 재검토**
   - "추천 안 함" 원칙을 유지할 것인가?
   - 시장 기대와 철학 사이의 트레이드오프

### 7.3 Go/No-Go 판단 기준

```
[Go 조건]
- 100명 베타 사용자 중 20%+ 유료 전환 의사
- 월 5회 이상 결정 기능 사용
- NPS 40+

[No-Go 시 대안]
- 오픈소스로 전환 (커뮤니티 빌딩)
- B2B 피벗 (코칭/상담 도구)
- 기술 스택 활용 다른 제품 개발
```
