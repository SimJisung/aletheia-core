## Phase 0: 문제 정의 (1주)

### 현재 상태의 근본 문제

```
[시장 적합성 문제]
"누가 이 제품을 원하는가?"에 대한 답이 없음

[가치 제안 문제]
"왜 돈을 내야 하는가?"에 대한 답이 없음
```

### 가설 재정립

| 기존 가설 | 검증 필요한 가설 |
|-----------|------------------|
| 사람들은 과거 기반 결정을 원한다 | 사람들은 **어떤 종류의** 결정에서 도움을 원하는가? |
| 확률 제시가 도움이 된다 | 확률 vs 추천 vs 질문 중 무엇이 실제로 도움되는가? |
| 가치 그래프가 유용하다 | 자신의 가치관을 **시각화**하고 싶어하는가? |

## Phase 1: 고객 발견 (2-4주)

### 1.1 타겟 세그먼트 검증 인터뷰

#### 인터뷰 대상 (각 5명씩, 총 20명)

| 세그먼트 | 가설 | 검증 질문 |
|----------|------|-----------|
| **A. 이직 고민자** | 커리어 결정에 데이터 기반 도움 필요 | "최근 이직 결정 어떻게 했나요?" |
| **B. 저널링 유저** | 이미 기록 습관 있음, 분석 원함 | "기록을 다시 읽어보나요? 왜?" |
| **C. 코치/상담사** | 클라이언트 이해 도구 필요 | "세션 전 준비를 어떻게 하나요?" |
| **D. 결정 장애** | 결정 자체가 고통스러움 | "결정을 미루는 이유가 뭔가요?" |

#### 인터뷰 스크립트 핵심 질문

```markdown
1. 최근 어려웠던 결정이 있다면?
2. 그 결정을 어떻게 내렸나요? (프로세스)
3. 결정 후 후회한 적 있나요? 왜?
4. 결정할 때 어떤 도움이 있으면 좋겠어요?
5. [프로토타입 시연 후] 이런 도구를 쓸 것 같나요?
6. 한 달에 얼마까지 낼 의향이 있나요?
```

#### 성공 기준

```
✅ 최소 1개 세그먼트에서:
   - 5명 중 3명 이상 "돈 낼 의향 있음"
   - 구체적인 사용 시나리오 언급
   - 기존 대안(ChatGPT 등)의 불만족 언급
```

### 1.2 경쟁 제품 심층 분석

| 분석 대상 | 분석 항목 |
|-----------|-----------|
| Reflect.app | 온보딩 플로우, 가격 정책, 리텐션 전략 |
| Day One | 무료→유료 전환 포인트 |
| Notion AI | AI 기능 사용 패턴 |
| ChatGPT | 의사결정 관련 프롬프트 분석 |

#### 실행 방법

```bash
# 1. 각 서비스 유료 구독 (1개월)
# 2. 실제 사용하며 기록
# 3. 리뷰/평점 분석 (App Store, ProductHunt)
# 4. Reddit/Twitter 사용자 피드백 수집
```

## Phase 2: 가치 제안 재정의 (2주)

### 2.1 "Jobs to be Done" 프레임워크 적용

#### 현재 JTBD (불명확)

```
"과거의 나를 기반으로 결정하고 싶다"
→ 너무 추상적, 행동으로 연결 안 됨
```

#### 재정의 후보

| JTBD | 타겟 | 경쟁 대안 | 차별화 포인트 |
|------|------|-----------|---------------|
| **A.** "중요한 결정에서 후회를 줄이고 싶다" | 이직/이사 고민자 | 친구, 코치, ChatGPT | 나의 패턴 기반 예측 |
| **B.** "내가 진짜 원하는 게 뭔지 알고 싶다" | 자기 탐색 중인 사람 | 심리 검사, 코칭 | 지속적 데이터 축적 |
| **C.** "클라이언트를 더 깊이 이해하고 싶다" | 코치/상담사 | 수기 노트, 설문 | 자동화된 가치 분석 |

### 2.2 One-Liner 테스트

#### A/B 테스트할 메시지

```markdown
Version A (기존):
"과거의 나를 기반으로 현재 결정의 적합도를 확률로 제시"

Version B (후회 중심):
"중요한 결정, 후회하기 전에 나의 패턴을 확인하세요"

Version C (자기 이해 중심):
"당신의 선택이 말해주는 진짜 가치관을 발견하세요"

Version D (실용 중심):
"이직, 이사, 관계 결정 - AI가 당신의 과거 선택을 분석합니다"
```

#### 테스트 방법

```
1. 랜딩 페이지 4개 버전 제작 (Carrd, Framer)
2. 각각 광고 집행 ($50/버전)
3. 이메일 수집 전환율 측정
4. 가장 높은 버전 채택
```

### 2.3 핵심 가치 3개로 압축

```markdown
## 제안: "3가지 약속"

1. **나다운 선택** - AI가 추천하지 않습니다. 당신의 패턴을 보여드릴 뿐.
2. **후회 예측** - 과거 비슷한 상황에서 어떤 선택이 후회로 이어졌는지.
3. **가치관 지도** - 8가지 축으로 나의 우선순위를 시각화.
```

## Phase 3: MVP 리포지셔닝 (2-4주)

### 3.1 "Quick Win" 기능 추가

#### 문제: 가치 체감까지 너무 오래 걸림

```
현재: 수십 개 fragment 입력 후에야 의미 있는 결과
목표: 첫 세션에서 즉시 가치 체감
```

#### 해결책: "빠른 가치 분석" 기능

```kotlin
// 새 API 엔드포인트 제안
POST /api/v1/quick-analysis

// 요청: 최근 중요했던 결정 3개만 입력
{
  "decisions": [
    { "situation": "이직 제안을 거절함", "feeling": "후회됨" },
    { "situation": "새 프로젝트 제안을 수락함", "feeling": "만족" },
    { "situation": "주말 약속을 취소함", "feeling": "후회됨" }
  ]
}

// 응답: 즉시 가치 패턴 분석
{
  "primary_value": "AUTONOMY",
  "insight": "당신은 자율성을 중요시하지만, 관계를 희생할 때 후회하는 경향이 있습니다.",
  "suggestion": "더 정확한 분석을 위해 생각을 기록해보세요."
}
```

### 3.2 온보딩 플로우 재설계

```
[현재]
회원가입 → 빈 화면 → "생각을 입력하세요" → 이탈

[개선안]
회원가입
  → 빠른 가치 분석 (3개 결정 입력, 2분)
  → 즉시 가치 프로필 제시
  → "더 정확하게 알고 싶다면 계속 기록하세요"
  → 가치 그래프 미리보기 (데이터 부족 표시)
```

### 3.3 가치 그래프 시각화 (필수)

```
[구현 필요]
- D3.js 또는 Chart.js 기반 레이더 차트
- 8축 가치 시각화
- 시간에 따른 변화 애니메이션
- 모순/갈등 하이라이트
```

#### 와이어프레임

```
        GROWTH
           │
   MEANING ┼─────── STABILITY
          ╱│╲
         ╱ │ ╲
   HEALTH  │  FINANCIAL
           │
   AUTONOMY ─── RELATIONSHIP
           │
      ACHIEVEMENT

[범례]
● 강한 가치 (상위 3개)
○ 약한 가치
━ 갈등 관계 (빨간 선)
```

## Phase 4: 시장 검증 (4-8주)

### 4.1 랜딩 페이지 + 대기자 명단

```markdown
## 구성 요소

1. 헤드라인 (테스트 승자)
2. 3가지 핵심 가치
3. 빠른 데모 (인터랙티브)
4. 가격 미리보기 ($9.99/월)
5. 대기자 등록 (이메일)
6. FAQ
```

#### 성공 지표

| 지표 | 목표 | 의미 |
|------|------|------|
| 방문→등록 전환율 | 10%+ | 메시지 공감 |
| 등록→유료 전환 의향 | 30%+ | 가격 수용성 |
| NPS (베타 후) | 40+ | 제품 만족도 |

### 4.2 클로즈드 베타 운영

#### 베타 유저 모집

```
목표: 50명
소스:
  - 대기자 명단 상위
  - 타겟 세그먼트 인터뷰 참여자
  - 관련 커뮤니티 (생산성, 저널링)
```

#### 베타 기간 수집 데이터

| 지표 | 측정 방법 | 목표 |
|------|-----------|------|
| DAU/MAU | 분석 도구 | 30%+ |
| Fragment 생성 빈도 | DB 쿼리 | 주 3회+ |
| Decision 사용률 | DB 쿼리 | 월 2회+ |
| 피드백 제출률 | DB 쿼리 | 50%+ |
| 인터뷰 인사이트 | 주간 콜 | 정성 데이터 |

### 4.3 가격 검증

#### Van Westendorp 가격 민감도 조사

```markdown
베타 유저에게 질문:

1. 이 제품이 너무 비싸서 구매하지 않을 가격은?
2. 비싸지만 구매를 고려할 가격은?
3. 적당하다고 느끼는 가격은?
4. 너무 싸서 품질이 의심되는 가격은?
```

#### 가격 테스트 방안

```
그룹 A: $4.99/월 제시
그룹 B: $9.99/월 제시
그룹 C: $14.99/월 제시

→ 전환율 비교하여 최적 가격 결정
```

## Phase 5: 피벗 또는 스케일 결정 (2주)

### 5.1 Go/No-Go 체크리스트

```markdown
## Go (스케일업) 조건 - 3개 이상 충족 시

☐ 베타 유저 50명 중 15명+ 유료 전환 의사 (30%)
☐ 주간 활성 사용률 40%+
☐ NPS 40+
☐ 유기적 추천 발생 (word-of-mouth)
☐ 명확한 타겟 세그먼트 1개 확정

## Pivot 조건 - 2개 이상 해당 시

☐ 유료 전환 의사 10% 미만
☐ 2주 후 리텐션 20% 미만
☐ "ChatGPT로 충분하다" 피드백 다수
☐ 핵심 기능 사용률 저조
```

### 5.2 피벗 옵션

#### Option A: B2B 피벗 (코칭 도구)

```markdown
타겟: 라이프 코치, 커리어 상담사
가격: $49-99/월/seat
기능 변경:
  - 클라이언트 관리 대시보드
  - 세션 전 분석 리포트
  - 가치 프로필 공유
```

#### Option B: 니치 집중 (이직 결정)

```markdown
타겟: 이직 고민 중인 직장인
가격: $19.99/월 또는 1회 $49
기능 변경:
  - 이직 결정 전문 플로우
  - 커리어 가치 분석
  - 회사 문화 적합도 평가
```

#### Option C: 오픈소스 전환

```markdown
수익 모델: 호스팅 서비스 + 엔터프라이즈
가격: Self-host 무료, Cloud $9.99/월
이점:
  - 커뮤니티 빌딩
  - 기술 인지도 확보
  - 기여자 확보
```

## 결론

```
시장 적합성과 가치 제안 강화의 핵심은:

1. 가정이 아닌 실제 고객 목소리에서 시작
2. "누구의 어떤 문제"를 푸는지 명확히 정의
3. 빠르게 가치를 체감할 수 있는 기능 추가

기술은 이미 충분합니다.
부족한 것은 "누가 왜 돈을 내는가"에 대한 검증입니다.
```
